**1. Create `src/lib/dashboard/types.ts`:**

Define the data types for the dashboard. Per D-02, each client card needs: name, company, current process number, gate status, running job indicator.

```typescript
import type { PhaseNumber } from '@/lib/database/enums'

export type DashboardClient = {
  id: string
  name: string
  company: string
  current_phase_number: PhaseNumber
  status: 'active' | 'archived'
  updated_at: string
  created_at: string
  // Enrichment from joins
  current_process_number: number | null      // highest active/completed process in current phase
  gate_status: 'pending' | 'evaluating' | 'approved' | 'rejected' | null
  has_running_job: boolean
  phase_started_at: string | null            // when client entered current phase (for bottleneck calc)
}

export type PhaseColumn = {
  phase_number: PhaseNumber
  phase_name: string
  clients: DashboardClient[]
}

export type DashboardData = {
  columns: PhaseColumn[]
  stuckClients: DashboardClient[]
}
```

**2. Create `src/lib/dashboard/constants.ts`:**

Per D-05, configurable bottleneck thresholds. Default: 7 days per phase.

```typescript
import type { PhaseNumber } from '@/lib/database/enums'

/** Bottleneck threshold in days per phase. Clients in a phase longer than this are flagged. */
export const BOTTLENECK_THRESHOLDS: Record<PhaseNumber, number> = {
```

**3. Create `src/lib/dashboard/queries.ts`:**

Single async function `fetchDashboardData` that:
- Uses `createClient()` from `@/lib/supabase/server`
- Fetches all clients (or only active, depending on `showArchived` param): `supabase.from('clients').select('id, name, company, current_phase_number, status, updated_at, created_at')`
- Fetches all phases for those clients: `supabase.from('phases').select('id, client_id, phase_number, status, started_at')`
- Fetches latest quality_gates per client (for current phase): `supabase.from('quality_gates').select('id, client_id, gate_number, status')`
- Fetches running squad_jobs: `supabase.from('squad_jobs').select('id, client_id, status').in('status', ['queued', 'running'])`
- Enriches each client into `DashboardClient` by:
  - Finding the phase row matching `client.current_phase_number` to get `phase_started_at`
  - Finding the gate for that phase to get `gate_status`
  - Checking if any running job belongs to that client for `has_running_job`
  - Finding the highest process number in the current phase that is active or completed for `current_process_number`
- Also fetch processes: `supabase.from('processes').select('id, client_id, process_number, status, phase_id')` to determine `current_process_number`
- Groups clients into 5 `PhaseColumn` objects
- Calculates stuck clients by comparing `phase_started_at` against `BOTTLENECK_THRESHOLDS` and `new Date()`
- Returns `DashboardData`

Function signature: `export async function fetchDashboardData(showArchived: boolean): Promise<DashboardData>`
