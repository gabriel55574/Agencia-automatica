    **Step 2: Create cost query module** (`src/lib/costs/queries.ts`):

    This module contains all Supabase queries for cost aggregation. Uses `createClient()` from `@/lib/supabase/server` (Server Component reads).

    ```typescript
    import { createClient } from '@/lib/supabase/server'
    import type { CostBreakdownRow, MonthlyCostSummary, BudgetStatus } from './types'
    import { BUDGET_THRESHOLDS } from './constants'
    ```

    **`fetchMonthlyCostBreakdown(month: string)`** -- month format: "2026-04"
    - Query: `squad_jobs` WHERE status='completed' AND token_count IS NOT NULL AND completed_at within the given month
    - JOIN with `clients` to get client_name
    - GROUP BY client_id, aggregate: COUNT(*) as run_count, SUM(token_count) as total_tokens, SUM(estimated_cost_usd) as total_cost
    - Also query previous month for trend calculation
    - Return: `CostBreakdownRow[]` sorted by total_cost DESC
    - Implementation note: Since Supabase JS client doesn't support GROUP BY, use an RPC function OR fetch raw rows and aggregate in TypeScript. For simplicity and to avoid requiring another migration, aggregate in TypeScript:
      1. Fetch all completed jobs for the month with `token_count IS NOT NULL` via `.not('token_count', 'is', null)`
      2. Fetch clients for those client_ids
      3. Group and sum in TypeScript using a Map
      4. For previous month: repeat the query with previous month range
    - Use date range filtering: `.gte('completed_at', monthStart).lt('completed_at', nextMonthStart)` where monthStart = `${month}-01T00:00:00Z` and nextMonthStart is calculated by adding 1 month

    **`fetchMonthlyCostSummary()`** -- current month only, for dashboard widget
    - Same aggregation pattern as above but for current month
    - Return: `MonthlyCostSummary` with total_cost, total_tokens, top 5 clients, previous_month_total
    - top_clients: take first 5 from the sorted-by-cost breakdown

    **`fetchProcessBudgetUsage(clientId: string)`** -- for budget bars on client profile
    - Query processes WHERE client_id = clientId AND token_budget IS NOT NULL
    - For each process with a budget, query SUM of token_count from squad_jobs WHERE process_id = process.id AND status = 'completed'
    - Return: `Record<string, { budget: number; used: number; status: BudgetStatus }>` keyed by process_id
    - BudgetStatus logic: used/budget < 0.8 = 'under', >= 0.8 and < 1.0 = 'approaching', >= 1.0 = 'exceeded'
