<objective>
Translate all server action user-facing error/success messages and email notification templates from English to Brazilian Portuguese. This is Wave 2 because email templates may reference UI terminology that should be consistent with the translations made in Wave 1. After this plan, server-side messages returned to the operator and email notifications sent by the system are all in PT-BR.
</objective>

<threat_model>
**Security Assessment:** Low security surface. This plan modifies error messages in server actions and HTML content in email templates. No authentication changes, no new endpoints, no input validation logic changes.
- **Threats identified:** None. Error message text changes do not affect security posture.
- **ASVS L1 relevance:** Verify that escapeHtml() in templates.ts remains intact after text changes (XSS prevention).
</threat_model>

<action>
Translate ALL user-facing error and success messages returned by server actions. These appear as toast notifications or form validation errors in the UI.

clients.ts:
- "Invalid source client ID" → "ID do cliente de origem invalido"
- "Source client not found" → "Cliente de origem nao encontrado"
- Any "Client created successfully" → "Cliente criado com sucesso"
- Any "Client updated successfully" → "Cliente atualizado com sucesso"
- Any "Client archived" → "Cliente arquivado"
- All other return { error: "..." } strings → translate to PT-BR

gates.ts:
- "Select at least one process to rework" → "Selecione pelo menos um processo para retrabalho"
- "Invalid gate or client ID" → "ID de gate ou cliente invalido"
- "Invalid input" → "Entrada invalida"
- Any success messages → translate

budget.ts:
- "Invalid input" → "Entrada invalida"
- "Invalid process ID" → "ID de processo invalido"
- Any budget success/error messages → translate

templates.ts:
- "Invalid template ID" → "ID de template invalido"
- "Failed to load templates" → "Falha ao carregar templates"
- "Invalid process number" → "Numero de processo invalido"
- "Invalid input" → "Entrada invalida"

squad.ts:
- "Invalid template ID" → "ID de template invalido"
- "Template not found" → "Template nao encontrado"
- "Invalid input" → "Entrada invalida"
- Any squad execution messages → translate

gate-review.ts:
- "Failed to fetch completed processes" → "Falha ao buscar processos concluidos"
- "Failed to fetch squad job outputs" → "Falha ao buscar outputs do squad"
- "No completed process outputs found for this phase" → "Nenhum output de processo concluido encontrado para esta fase"
- "Failed to create gate review job" → "Falha ao criar tarefa de revisao do gate"
- "Failed to create gate review record" → "Falha ao criar registro de revisao do gate"

pipeline-reset.ts:
- "Invalid client ID" → "ID de cliente invalido"
- Any reset success/error messages → translate

Do NOT translate:
- console.log/console.error messages
- Technical error details sent to Sentry
- Zod schema field names (these are internal)
</action>
