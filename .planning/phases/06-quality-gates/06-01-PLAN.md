    Per D-09, create src/lib/gates/review-schema.ts:

    ```typescript
    import { z } from 'zod'

    export const GateReviewVerdictItemSchema = z.object({
      checklist_id: z.string().min(1),
      label: z.string().min(1),
      verdict: z.enum(['pass', 'fail']),
      evidence: z.string(), // quote or reference from actual output
      notes: z.string(),    // auditor explanation
    })

    export const GateReviewVerdictSchema = z.object({
      gate_number: z.number().int().min(1).max(4),
      overall: z.enum(['pass', 'fail', 'partial']),
      items: z.array(GateReviewVerdictItemSchema).min(1),
      summary: z.string().min(1), // overall assessment
    })

    export type GateReviewVerdict = z.infer<typeof GateReviewVerdictSchema>
    export type GateReviewVerdictItem = z.infer<typeof GateReviewVerdictItemSchema>
    ```

    5. **Evaluation Instructions:** "For each checklist item: Evaluate whether the phase outputs satisfy this requirement. Cite specific text from the outputs as evidence. If an item cannot be verified from the outputs, mark it as FAIL with an explanation in notes."

    If gateNumber is invalid (getGateChecklist returns null), throw an Error: "No checklist found for gate {gateNumber}".

|----------|-------------|
| CLI output -> verdict parse | Untrusted AI output crosses into application data |
| Checklist content -> prompt | Static content embedded in prompts sent to Claude CLI |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-01 | T (Tampering) | review-schema.ts | mitigate | Zod safeParse on all CLI output before storing as verdict -- reject malformed JSON |
| T-06-02 | I (Information Disclosure) | review-prompt.ts | accept | Prompt contains client outputs -- same security boundary as squad prompts (T-05-02 pattern) |
