    **Part A: Create src/lib/actions/gate-review.ts** (per D-03, D-07)

    This Server Action follows the same pattern as confirmSquadRun in src/lib/actions/squad.ts but with gate-specific logic.

    ```typescript
    'use server'

    import { z } from 'zod'
    import { revalidatePath } from 'next/cache'
    import { createClient } from '@/lib/supabase/server'
    import { createAdminClient } from '@/lib/supabase/admin'
    import { buildReviewPrompt } from '@/lib/gates/review-prompt'
    import type { ActionResult } from './clients'

    const runGateReviewSchema = z.object({
      gateId: z.string().uuid(),
      clientId: z.string().uuid(),
      gateNumber: z.number().int().min(1).max(4),
      phaseId: z.string().uuid(),
    })
    ```

    The `runGateReview` function:
    1. Auth check: `createClient().auth.getUser()` -- return `{ error: 'Unauthorized' }` if no user
    2. Validate input with `runGateReviewSchema.safeParse`
    3. Query all processes in the phase with status='completed' from the processes table:
       ```typescript
       const { data: completedProcesses } = await admin
         .from('processes')
         .select('id, process_number, name')
         .eq('phase_id', input.data.phaseId)
         .eq('status', 'completed')
       ```
    4. For each completed process, fetch the most recent completed squad_job output:
       ```typescript
       const { data: jobs } = await admin
         .from('squad_jobs')
         .select('output, process_id')
         .eq('phase_id', input.data.phaseId)
         .eq('status', 'completed')
         .not('output', 'is', null)
         .not('squad_type', 'eq', 'gate_review')
       ```
    5. Build phase outputs array: `{ processName, processNumber, output }[]` by matching jobs to completedProcesses via process_id
    6. Call `buildReviewPrompt(gateNumber, phaseOutputs)` to get the full prompt
    7. INSERT into squad_jobs with:
       - client_id, phase_id from input
       - process_id: null (gate reviews are not process-specific)
       - squad_type: 'gate_review'
       - status: 'queued'
       - cli_command: the full prompt string
    8. INSERT into gate_reviews with:
       - gate_id, client_id from input
       - verdict: `{}` (placeholder -- worker fills this)
       - raw_output: '' (placeholder -- worker fills this)
       - status: 'running'
       - squad_job_id: the id from the squad_jobs insert
    9. `revalidatePath('/clients/${clientId}')`
    10. Return `{ success: true, jobId: insertedJobId }`

    Export return type: `Promise<{ success: true; jobId: string } | { error: string }>`

    **Part B: Extend src/worker/job-runner.ts** (per D-10)

    In the `proc.on('close', ...)` handler inside `runJob()`, after the existing structured output parsing block, add a gate_review-specific branch:

    ```typescript
    // After existing: if (parsedContent !== null && job.process_id) { ... }

    // Gate review jobs: parse verdict and store in gate_reviews table
    if (job.squad_type === 'gate_review') {
      const parsedContent = parseCliOutput(stdoutBuffer)
      if (parsedContent !== null) {
        const { GateReviewVerdictSchema } = await import('../lib/gates/review-schema')
        const verdictResult = GateReviewVerdictSchema.safeParse(parsedContent)

        // Update gate_reviews row linked to this job
        if (verdictResult.success) {
          await supabase
            .from('gate_reviews')
            .update({
              verdict: verdictResult.data,
              raw_output: stdoutBuffer,
              status: 'completed',
            })
            .eq('squad_job_id', job.id)
        } else {
          // Parse failed -- store raw output, mark as failed
          process.stdout.write(
            `[worker] Gate review verdict parse failed for job ${job.id}: ${verdictResult.error.message}\n`
          )
          await supabase
            .from('gate_reviews')
            .update({
              verdict: { parse_error: verdictResult.error.message, raw: parsedContent },
              raw_output: stdoutBuffer,
              status: 'failed',
            })
            .eq('squad_job_id', job.id)
        }
      } else {
        // CLI output not parseable at all
        await supabase
          .from('gate_reviews')
          .update({
            verdict: { parse_error: 'CLI output could not be parsed' },
            raw_output: stdoutBuffer,
            status: 'failed',
          })
          .eq('squad_job_id', job.id)
      }
    }
    ```

    IMPORTANT: Use dynamic import for `GateReviewVerdictSchema` since the worker uses relative imports and runs outside Next.js bundler. The import path should be `'../lib/gates/review-schema'` (relative, not @/ alias).

    The existing structuredOutput logic for process jobs remains unchanged. The gate_review branch only fires when `job.squad_type === 'gate_review'`.

    Also handle the failure case: when the gate_review job fails (not success branch), update gate_reviews status to 'failed':
    ```typescript
    // In the !success branch, before handleFailure:
    if (job.squad_type === 'gate_review') {
      await supabase
        .from('gate_reviews')
        .update({
          verdict: { error: 'Job failed', stderr: stderrBuffer.slice(0, 2000) },
          raw_output: stdoutBuffer,
          status: 'failed',
        })
        .eq('squad_job_id', job.id)
    }
    ```
