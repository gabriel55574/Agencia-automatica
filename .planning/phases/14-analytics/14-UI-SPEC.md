### Semantic Colors for Analytics

| Color | Hex Reference | Reserved For |
|-------|--------------|--------------|
| Emerald | `emerald-500` | Gate approval rate above 80% (healthy), positive trends on charts |
| Amber | `amber-500` | Gate approval rate 50-79% (moderate), phase duration above threshold |
| Red | `red-500` | Gate approval rate below 50% (poor), bottleneck indicator bars |
| Blue | `blue-500` | Primary chart line/bar fill color, active date range filter |
| Zinc | `zinc-300` (light) / `zinc-600` (dark) | Chart grid lines, axis labels, inactive filter buttons |

Accent reserved for: Active date range filter button, page heading text, metric card primary values.

Source: Consistent with existing semantic color patterns from KanbanClientCard, BottleneckAlert, Phase 12 UI-SPEC.

### Date Range Filter Bar

Position: Centered below page subtitle, above the charts grid.

- shadcn `ToggleGroup` with `type="single"` and `variant="outline"`
- 4 options: "7d", "30d", "90d", "All Time"
- Default selection: "30d"
- Active button: `bg-zinc-900 text-white` (dark fill, white text)
- Inactive buttons: `bg-white text-zinc-600 border-zinc-200`
- Button size: `h-8 px-3 text-xs font-medium`
- Gap between buttons: `0` (connected group, first/last rounded)

### Phase Performance Section (ANLY-01)

- Card component with `CardHeader` + `CardContent`
- Header: "Phase Performance" at `text-base font-semibold text-zinc-900`
- Subtitle: "Average days per phase" at `text-xs text-zinc-500`
- Chart: Recharts `BarChart` with horizontal orientation (`layout="vertical"`)
  - Y-axis: Phase names from PHASE_NAMES enum (Portuguese labels)
  - X-axis: Days (numeric)
  - Bar fill: `blue-500` default
  - Slowest phase bar: `red-500` fill (bottleneck indicator)
  - Bar height: 24px (`barSize={24}`)
  - Value label: `text-xs font-mono text-zinc-600` rendered at end of each bar, format "X.Xd"
- Tooltip: Shows full phase name + exact days on hover
- If no completed phases: show empty state

### Gate Approval Rates Section (ANLY-02)

- Card component with `CardHeader` + `CardContent`
- Header: "Gate Approval Rates" at `text-base font-semibold text-zinc-900`
- Subtitle: "First-pass approval rate per gate" at `text-xs text-zinc-500`
- Display: 4 rows (one per gate), each with:
  - Gate label: `text-sm text-zinc-700` "Gate 1 (Diagnostico)"
  - Percentage: `text-sm font-mono font-semibold` with semantic color
  - Horizontal progress bar: `h-2` fill color based on rate
    - 80%+: `bg-emerald-500` (healthy)
    - 50-79%: `bg-amber-500` (moderate)
    - Below 50%: `bg-red-500` (poor)
  - Badge next to percentage: `Badge` with semantic variant
    - 80%+: emerald badge "Healthy"
    - 50-79%: amber badge "Moderate"
    - Below 50%: red badge "Poor"
- Count label below bar: `text-xs text-zinc-400` "X of Y gates passed first try"
- If no evaluated gates: show empty state

### Client Lifecycle Section (ANLY-03)

- Card component with `CardHeader` + `CardContent`
- Header: "Client Lifecycle" at `text-base font-semibold text-zinc-900`
- Subtitle: "Time from intake to Phase 5 completion" at `text-xs text-zinc-500`
- Primary metric: Average days displayed as `text-3xl font-semibold font-mono text-zinc-900`
- Completed count: `text-sm text-zinc-500` "{N} clients completed full cycle"
- Client list: vertical list of completed clients
  - Each row: client name (`text-sm text-zinc-700`) + duration (`text-sm font-mono text-zinc-500`, format "Xd")
  - Max 10 entries shown, with "and X more" text if overflow
  - Client names are NOT links (analytics page is aggregate view)
- If no completed clients: show empty state

### Trends Section (ANLY-04)

- Card component with `CardHeader` + `CardContent`
- Header: "Trends" at `text-base font-semibold text-zinc-900`
- Subtitle: "Monthly performance over time" at `text-xs text-zinc-500`
- Chart: Recharts `LineChart`
  - X-axis: months (abbreviated, e.g., "Jan", "Feb", "Mar")
  - Y-axis: metric value (auto-scaled)
  - Two lines:
    - "Avg Phase Duration" — `blue-500` stroke, `strokeWidth={2}`
    - "Gate Approval Rate" — `emerald-500` stroke, `strokeWidth={2}`
  - Legend: Recharts `Legend` component below chart
  - Grid: `CartesianGrid strokeDasharray="3 3"` with `zinc-200` stroke
  - Dots: `dot={false}` for clean lines, `activeDot={{ r: 4 }}` on hover
  - Tooltip: Recharts `Tooltip` with custom styling matching card background
- If insufficient data (fewer than 2 months): show "Not enough data for trends. Trends appear after 2+ months of operation."

### Charts Grid Layout

- Desktop (`lg`+): 2-column grid with `gap-6`
  - Row 1: Phase Performance (left), Gate Approval Rates (right)
  - Row 2: Client Lifecycle (left), Trends (right)
- Tablet (`md`): 2-column grid maintained, same layout
- Mobile (`sm`): Single column stack, all 4 cards full width

Tailwind: `grid grid-cols-1 lg:grid-cols-2 gap-6`

## Interaction Specifications

### Date Range Filter (ANLY-04)

- Toggle group with 4 mutually exclusive options
- Default: "30d" selected on page load
- On change: filters all 4 chart sections simultaneously
- State managed via React `useState` in client component (no URL params for filter -- filter is ephemeral)
- Filter logic: include only data where the relevant timestamp falls within the selected period from today
  - "7d": last 7 days
  - "30d": last 30 days
  - "90d": last 90 days
  - "All Time": no filter

### Chart Hover Behavior

- Recharts `Tooltip` shows on hover over bars/lines
- Tooltip styling: `bg-white border border-zinc-200 shadow-sm rounded-md p-2 text-sm`
- Tooltip disappears when mouse leaves chart area
- No click interactions on chart elements in v1

### Navigation

- Clicking "Dashboard" in header navigates to `/`
- Clicking "Analytics" in header navigates to `/analytics`
- Active page link is visually distinguished (semibold + darker color)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Date range filter | `ToggleGroup` with `aria-label="Date range filter"`, each item has descriptive label |
| Chart accessibility | Each Recharts chart wrapped in a container with `role="img"` and `aria-label` describing the data |
| Phase performance chart | `aria-label="Bar chart showing average days per phase: {phase1} {days1}, {phase2} {days2}, ..."` |
| Gate approval chart | `aria-label="Gate approval rates: Gate 1 {rate1}%, Gate 2 {rate2}%, ..."` |
| Trends chart | `aria-label="Trend chart showing monthly averages for phase duration and gate approval rate"` |
| Empty states | Empty state messages are visible text (no hidden content) |
| Color-only information | Gate approval status conveyed by both color bar AND text label (percentage + badge text) |
| Metric values | Numeric values are plain text, readable by screen readers |
| Navigation | Active page indicated by both `font-semibold` styling and `aria-current="page"` attribute |
