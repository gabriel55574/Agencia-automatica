## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (light) / `#0a0a0a` (dark) | Page background (`bg-background`) |
| Secondary (30%) | `zinc-50` (light) / `zinc-900` (dark) | Dashboard shell, card surfaces (`bg-zinc-50`, `Card` component) |

### Semantic Colors for Cost Tracking

| Color | Hex Reference | Reserved For |
|-------|--------------|--------------|
| Emerald | `emerald-100/800` (bg/text) | Under-budget status badge, healthy spend indicator |
| Amber | `amber-100/800` (bg/text) | Approaching-budget warning badge (80-99% of budget), budget alert banner |
| Red | `red-100/800` (bg/text) | Over-budget badge (100%+ of budget), exceeded budget alert |
| Blue | `blue-100/800` (bg/text) | Active/running spend indicator (consistent with existing running-job badge) |
| Zinc | `zinc-100/600` (bg/text) | Neutral/no-budget-set badge |

Accent reserved for: Primary CTA buttons ("Set Budget", "View Costs"), sortable column active state headers, monthly total display value.

Source: Existing badge color patterns from KanbanClientCard, ActionSummaryBar, BottleneckAlert.

## Layout Specifications

### COST-04: Dashboard Cost Summary Widget

Position: Below the `ActionSummaryBar`, above the Kanban board on the main dashboard (`src/app/(dashboard)/page.tsx`).

```
+--------------------------------------------------+
| Dashboard (text-2xl font-semibold)               |
+--------------------------------------------------+
| [BottleneckAlert]                                |
| [ActionSummaryBar]                               |
|                                                  |
| +----------------------------------------------+ |
| | Cost Summary              [View All Costs ->]| |
| |                                              | |
| | This month: $1,234.56        Apr 2026        | |
| |                                              | |
| | Top clients:                                 | |
| |   1. ClientA .............. $320.00           | |
| |   2. ClientB .............. $280.00           | |
| |   3. ClientC .............. $195.00           | |
| +----------------------------------------------+ |
|                                                  |
| [KanbanBoard]                                    |
+--------------------------------------------------+
```

- Card component with `CardHeader` + `CardContent`
- Header: "Cost Summary" at `text-base font-semibold text-zinc-900`
- "View All Costs" link button at `text-sm` aligned right in header
- Total spend: `text-2xl font-semibold text-zinc-900` for dollar amount, `font-mono` for numerical alignment
- Month label: `text-sm text-zinc-500`
- Top clients list: max 5 entries, each row with client name (left-aligned `text-sm text-zinc-700`) and cost (right-aligned `text-sm font-mono text-zinc-900`)
- If no data: show empty state (see Copywriting Contract below)

### COST-01: Run Cost Badge on Run Details

Position: Inline within the existing process row in `PipelineAccordion`, next to the job status badge.

```
Process 1: Market Research  [Active] [Running] [$0.45 | 12.3k tokens]
```

- `RunCostBadge` renders as a `Badge variant="outline"` with `text-xs font-mono`
- Format: `$X.XX | XX.Xk tokens` (dollar cost pipe-separated from token count)
- Only visible when the squad job has `status === 'completed'`
- Token count uses suffixes: values under 1000 show raw number, 1k-999k show `Xk`, 1M+ show `X.XM`
- Color: default zinc outline badge (matches process number badge style)

Route: `/costs`

```
+--------------------------------------------------+
| Cost Breakdown              [Month Selector v]   |
+--------------------------------------------------+
| April 2026 Total: $4,567.89                      |
+--------------------------------------------------+
| [By Client] [By Process]  tabs                   |
+--------------------------------------------------+

|------------|------|-----------|----------|-------|

| ClientB    |    8 | 980k      | $280.00  |  -3%  |

```

- Page heading: `text-2xl font-semibold text-zinc-900` "Cost Breakdown"

- Monthly total: `text-xl font-semibold text-zinc-900` with `font-mono` for the dollar amount
- Tab bar: shadcn `Tabs` component with two tabs: "By Client" (default), "By Process"

  - All numeric columns right-aligned with `text-right font-mono`

  - Active sort column header: `font-semibold text-zinc-900` with sort direction arrow icon (ArrowUpDown from Lucide)
  - Trend column: green text (`text-emerald-600`) for decrease, red text (`text-red-600`) for increase, zinc text for flat

  - Client name links to `/clients/[id]`
- Max width: `max-w-4xl` (wider than client detail pages to accommodate table)

### COST-03: Budget Setting and Alerts

**Budget Setting Dialog:**

Triggered by a "Set Budget" button on each process row in the pipeline accordion, or from the cost breakdown page via a per-row action.

```
+------------------------------------------+
| Set Token Budget                    [X]  |
|                                          |
| Process: Market Research (P1)            |
|                                          |
| Token Limit                             |
| [___100,000___]                          |
| Estimated cost: ~$2.50                   |
|                                          |
| [Discard Changes]           [Save Budget]|
+------------------------------------------+
```

- Dialog title: `text-lg font-semibold`
- Process name shown as `text-sm text-zinc-500` subtitle
- Input: shadcn `Input` with `type="number"`, right-aligned text, `font-mono`
- Estimated cost shown below input: `text-xs text-zinc-400`, calculated at $0.025 per 1k tokens (configurable constant)
- Discard Changes: `Button variant="ghost"` with label "Discard Changes"
- Save: `Button variant="default"` with label "Save Budget"
- Validation: minimum 1000 tokens, must be a whole number
- On save: toast via `sonner` -- "Budget set for Market Research: 100,000 tokens"

**Budget Bar (inline on process row):**

When a budget is set, show a progress bar below the process name in the pipeline accordion.

```
Process 1: Market Research  [Active] [Running] [$0.45 | 12.3k tokens]
[||||||||||||--------] 62% of 100k budget
```

- shadcn `Progress` component with custom color thresholds:
  - 0-79%: default zinc fill (normal)
  - 80-99%: amber fill (`bg-amber-500`) -- approaching budget
  - 100%+: red fill (`bg-red-500`) -- over budget (bar caps at 100% visually, text shows actual %)
- Percentage label: `text-xs text-zinc-500 font-mono` right-aligned, e.g., "62% of 100k"
- Bar height: `h-2` (8px, matching spacing scale)
- Full width within the process row content area

**Budget Alert Banner:**

When any process in the current view has exceeded its budget, show an alert banner (follows BottleneckAlert pattern).

```
+--------------------------------------------------+
| [!] 2 process(es) exceeded their token budget     |
|   - ClientA > P1 Market Research: 125% (125k/100k)|
|   - ClientB > P7 G-STIC Plan: 110% (55k/50k)     |
+--------------------------------------------------+
```

- Same pattern as `BottleneckAlert`: colored background with border and icon
- Approaching (80-99%): `bg-amber-50 border-amber-200`, `AlertTriangle` icon in `text-amber-600`
- Exceeded (100%+): `bg-red-50 border-red-200`, `AlertTriangle` icon in `text-red-600`
- Each item links to the client detail page
- Only shown on the `/costs` page and the dashboard

### Number Formatting Rules

| Data Type | Format | Example |
|-----------|--------|---------|
| Dollar cost | `$X,XXX.XX` (2 decimal places, comma thousands) | `$1,234.56` |
| Token count (compact) | Suffix: raw if <1k, `Xk` if 1k-999k, `X.XM` if 1M+ | `850`, `12.3k`, `1.2M` |
| Token count (full) | Comma-separated | `100,000` |
| Percentage | `X%` (no decimals, round) | `62%` |
| Trend | `+X%` / `-X%` / `0%` (sign prefix, no decimals) | `+5%`, `-3%`, `0%` |
| Month display | `MMMM YYYY` | `April 2026` |

All monetary and numeric values render in `font-mono` (Geist Mono via `--font-geist-mono`).

## Interaction Specifications

### Sort Behavior (COST-02 Table)

- Default sort: Cost column, descending (highest spender first)
- Click column header to sort ascending; click again for descending; click a third time to return to default
- Sort state managed via URL search params (`?sort=cost&dir=desc`) for shareable URLs
- Visual indicator: Lucide `ArrowUp` / `ArrowDown` icon (14px, `h-3.5 w-3.5`) next to active sort column header text

### Month Navigation (COST-02)

- shadcn `Select` dropdown showing current month by default
- Options: last 12 months in reverse chronological order
- On change: updates URL search param `?month=2026-04` and refetches data via RSC
- No "all time" option in v1 -- scope to single months

### Budget Dialog Flow (COST-03)

1. Operator clicks "Set Budget" on a process row
2. Dialog opens with process name pre-filled (read-only)
3. Operator enters token limit number
4. Estimated cost updates live as operator types (client-side calculation)
5. Operator clicks "Save Budget" -- Server Action writes to database
6. Dialog closes, toast confirms save
7. Budget bar appears on the process row

### Budget Removal (COST-03)

1. If a budget exists, the "Set Budget" button label changes to budget icon button (Lucide `Settings2`, `h-3.5 w-3.5`) with `aria-label="Edit token budget for {processName}"` and a `Tooltip` on hover showing "Edit Budget"
2. Inside the dialog, a "Remove Budget" ghost button appears at bottom-left
3. Clicking "Remove Budget" shows inline confirmation: "Remove budget for {processName}?"
4. Confirming removes the budget, closes dialog, shows toast: "Budget removed for {processName}"

### Dashboard Widget Interaction (COST-04)

- "View All Costs" link navigates to `/costs` with the current month pre-selected
- Client names in the top-5 list are clickable links to `/clients/[id]`
- Widget refreshes with page (RSC -- no client-side polling needed)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Budget progress bar | `aria-label="Token budget usage: {percent}% of {limit} tokens"`, `role="progressbar"`, `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax=100` |
| Sortable table headers | `aria-sort="ascending"` / `"descending"` / `"none"` on `<th>` elements |
| Budget alert banner | `role="alert"` for over-budget alerts |
| Cost values | `aria-label` on abbreviated token counts (e.g., `aria-label="12,300 tokens"` when displaying "12.3k") |
| Dialog | Handled by Radix Dialog (focus trap, Escape to close, aria-labelledby) |
| Color-only information | Budget status is conveyed by both color AND text label (percentage + "over budget" / "approaching") |
| Budget settings icon button | `aria-label="Edit token budget for {processName}"` on the Lucide `Settings2` icon button. Wrapped in shadcn `Tooltip` with content "Edit Budget" shown on hover/focus. |
