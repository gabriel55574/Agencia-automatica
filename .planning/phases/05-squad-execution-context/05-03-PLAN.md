## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client component -> Server Action | Button click triggers assembleSquadContext/confirmSquadRun |
| Server -> Client | Assembled context + prompt sent to client for preview display |
| DB -> UI | structured_output JSONB rendered as HTML (potential XSS vector) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Info Disclosure | PromptPreviewModal | accept | Prompt contains client briefing + prior outputs. Displayed only to authenticated operator. No cross-client risk since assembleSquadContext scopes to clientId. Solo operator model — acceptable. |
| T-05-11 | Tampering | StructuredOutputView | mitigate | Render structured_output fields as text nodes only (React escapes by default). Never use dangerouslySetInnerHTML. Pre block for raw output also text-only. |
| T-05-12 | Denial of Service | RunSquadButton | mitigate | Loading state disables button during assembleSquadContext call. Cannot spam-click to create multiple jobs. D-03 also hides button when job is in-flight. |
| T-05-13 | Repudiation | confirmSquadRun | accept | squad_jobs table records who triggered (via cli_command content) and when (created_at). Sufficient audit trail for solo operator. |
</threat_model>
