    export type ProcessOutput = {

    ): Promise<AssembledContext>

    Implementation details:
    1. Accept an optional supabaseClient parameter for dependency injection in tests.
       Default to `createAdminClient()` if not provided.
    2. Query `clients` for `briefing` and `current_phase_number` WHERE `id = clientId`.
    3. Query `squad_jobs` joined to `processes` joined to `phases`:
       - `squad_jobs.status = 'completed'`
       - `squad_jobs.output IS NOT NULL`
       - `phases.phase_number < client.current_phase_number` (D-04: exclude same-phase)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | output-parser.ts | mitigate | Two-level JSON.parse wrapped in try/catch; safeParse never throws; malformed output returns null, preserving raw in squad_jobs.output |
| T-05-02 | Info Disclosure | assembler.ts | mitigate | assembleContext always scopes queries with WHERE client_id = $1; no cross-client data leakage possible in single-client query |
| T-05-03 | Denial of Service | output-parser.ts | accept | safeParse on arbitrarily large CLI output is bounded by CLI session timeout (already enforced by Phase 4 worker). No additional DoS mitigation needed. |
| T-05-04 | Tampering | structured_output JSONB | mitigate | Only written after successful safeParse(); never stores unvalidated JSON in structured_output column |
</threat_model>
