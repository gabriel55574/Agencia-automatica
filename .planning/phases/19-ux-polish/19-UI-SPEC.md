## Color

## Component Inventory — Phase 19

### Skeleton Components (loading.tsx files)

| Route | Skeleton Layout | Key Visual Elements |
|-------|----------------|---------------------|
| `/` (dashboard) | 5-column Kanban grid | 5 column headers (h-6 w-24 pulse blocks) + 3 card placeholders per column (h-20 w-full pulse blocks) |
| `/clients` | Card grid | 6 card placeholders in 3x2 grid (h-32 w-full pulse blocks) |
| `/clients/[id]` | Tabbed profile | Header block (h-8 w-48) + 3 tab buttons (h-10 w-24 each) + content area (h-64 w-full) |
| `/analytics` | Chart area | 2 chart placeholders side by side (h-48 w-1/2) + summary row of 4 stat cards (h-20 w-1/4) |
| `/costs` | Data table | Table header row (h-10) + 8 table body rows (h-12 each) |
| `/templates` | List | 4 list item placeholders (h-16 w-full) |

**Skeleton style:** `animate-pulse` with `bg-[#E8EDED]` (gray-100) blocks on white `bg-white` cards. Rounded corners `rounded-md` (8px). No text, icons, or borders — shape-only placeholders matching actual content dimensions.

### EmptyState Component

Reusable component: `src/components/ui/empty-state.tsx`

**Props:**
| Prop | Type | Required | Description |
|------|------|----------|-------------|
| icon | LucideIcon | yes | Lucide icon component (48px, text-[#8A9999] gray-300) |
| title | string | yes | Heading text (heading-sm: 20px Montserrat 600) |
| description | string | yes | Body text (body-md: 16px Roboto 400, text-[#5C6E6E] gray-400) |
| actionLabel | string | no | CTA button text |
| actionHref | string | no | CTA button link destination |

**Layout:** Centered vertically and horizontally in parent. Flex column, items-center, gap-16px (md spacing). Icon at top, then title, then description, then CTA button (if provided). Max-width 320px for text block.

**CTA button style:** Electric Lime (#BFF205) background, Rich Black (#062222) text, rounded-md, px-16px py-8px. Uses existing shadcn Button component with `variant="default"` (maps to primary).

### Client Profile Tabs

**Component location:** `src/app/(dashboard)/clients/[id]/page.tsx` (restructured)

**Tab configuration:**
| Tab | Value | Default | Content Source |
|-----|-------|---------|----------------|
| Pipeline | "pipeline" | yes | Existing PipelineAccordion component |
| Outputs | "outputs" | no | Output list (moved from /clients/[id]/outputs) |
| Briefing | "briefing" | no | Client briefing data with edit CTA |

**Tab style:** Uses shadcn `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent` components. TabsList has `w-full` width. Active tab indicator uses default shadcn styling (sufficient — no custom accent override needed). Tab content area has `pt-24px` (lg spacing) padding.

**Briefing tab content:**
- Display fields: niche, target_audience, additional_context (read-only, body-md text)
- "Editar Briefing" button at bottom-right, links to `/clients/[id]/edit`
- Empty briefing fields show "Nao informado" in gray-300 italic

### Toast Integration Points

| Action | Component | Toast Type | Message |
|--------|-----------|------------|---------|
| Create client | Client form submit handler | success | "Cliente criado com sucesso" |
| Update client | Client edit form submit handler | success | "Cliente atualizado" |
| Archive client | Archive action in client profile | success | "Cliente arquivado" |
| Restore client | Restore action in client list | success | "Cliente restaurado" |
| Form validation | Client form validation catch | error | "Erro ao salvar: verifique os campos obrigatorios" |

**Toast implementation:** `import { toast } from 'sonner'` — use `toast.success()` for success, `toast.error()` for errors. Sonner is already mounted via `src/components/ui/sonner.tsx`. Default positioning and styling (no overrides).

## Visual Hierarchy

**Focal points per screen:**

| Screen | Primary Focal Point | Secondary |
|--------|-------------------|-----------|
| Dashboard (Kanban) | Phase columns with client cards | "Novo Cliente" CTA if empty |
| Client list | Client cards grid | "Novo Cliente" CTA if empty |
| Client profile | Active tab content | Tab navigation bar |
| Analytics | Chart visualizations | "Dados insuficientes" empty state if no data |
| Costs | Cost data table | Budget summary cards |
| Templates | Template list | "Criar Template" CTA if empty |

**Empty state visual weight:** Empty states should feel lightweight — the icon and CTA are the only "heavy" elements. Text is secondary (gray-400 body). The empty state should guide, not overwhelm.

## Accessibility Notes

- Skeleton screens use `aria-busy="true"` on the loading container and `role="status"` with `aria-label="Carregando..."` for screen readers
- Empty state icons are decorative (`aria-hidden="true"`) — the title provides the semantic meaning
- Tab navigation supports keyboard arrow keys (built into shadcn Tabs/Radix)
- Toast announcements use Sonner's built-in `role="status"` and `aria-live="polite"`
