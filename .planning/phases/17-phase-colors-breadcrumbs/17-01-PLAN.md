<objective>
Define a centralized PHASE_COLORS constant mapping each of the 5 pipeline phases to distinct color variants (base, light, dark, border), then apply those colors across Kanban column headers, pipeline accordion items, and phase status badges — replacing all hard-coded zinc/blue/green colors with phase-aware styling.
</objective>

<threat_model>
## Threat Model

**Attack surface:** This plan modifies only client-side UI constants and component styling. No user input is processed, no data is persisted, no API calls are made.

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-17-01 | XSS via phase color injection | N/A | Colors are compile-time constants in TypeScript, not user-supplied. No dynamic class generation from external input. |

**Residual risk:** None. This is a pure presentational change with no security surface.
</threat_model>

export const PHASE_COLORS = {
  1: { base: 'text-blue-500', light: 'bg-blue-500/10', dark: 'text-blue-700', border: 'border-blue-500' },
  2: { base: 'text-violet-500', light: 'bg-violet-500/10', dark: 'text-violet-700', border: 'border-violet-500' },
  3: { base: 'text-amber-500', light: 'bg-amber-500/10', dark: 'text-amber-700', border: 'border-amber-500' },

1. Add import: `import { PHASE_COLORS, type PhaseNumber } from '@/lib/database/enums'`

2. Update `PhaseStatusBadge` to accept `phaseNumber` prop and use phase-aware colors:

Replace the entire `PhaseStatusBadge` function:
```tsx
function PhaseStatusBadge({ status, phaseNumber }: { status: PhaseRow['status']; phaseNumber: PhaseNumber }) {
  const colors = PHASE_COLORS[phaseNumber]
  if (status === 'active') return <Badge className={`${colors.light} ${colors.dark} border-transparent`}>Active</Badge>
  if (status === 'completed') return <Badge className={`${colors.light} ${colors.dark} border-transparent`}>Completed</Badge>
  return <Badge variant="secondary">Pending</Badge>
}
```

For active and completed phases, the badge uses the phase's light background + dark text color. Pending phases remain neutral (secondary variant). This replaces the hard-coded blue-100/blue-700 and green-100/green-700.

3. Update the `PhaseStatusBadge` call in the component to pass `phaseNumber`:
```tsx
<PhaseStatusBadge status={phase.status} phaseNumber={phase.phase_number as PhaseNumber} />
```

The `border-l-[3px]` overrides the general `border` for the left side. The `PHASE_COLORS[...].border` applies the phase-specific color (e.g., `border-blue-500` for Phase 1).
