**Per D-08:** Create `src/components/documents/OutputPdfTemplate.tsx`.

This file exports a React component that returns a `@react-pdf/renderer` `<Document>`. It is NOT a regular React DOM component -- it uses the react-pdf primitives (Document, Page, Text, View, StyleSheet, Font).

**Important:** This component must NOT be imported at the top level of any Server Component or any file that gets server-rendered. It will only be used via dynamic import in the client-side OutputViewer. Add a comment at the top of the file: `// Client-side only -- import via dynamic import or lazy loading. Do NOT import in Server Components.`

**Props for OutputPdfTemplate:**
```typescript
interface OutputPdfTemplateProps {
  clientName: string
  processName: string
  phaseName: string
  date: string               // formatted date string
  structuredOutput: Record<string, unknown> | null
  rawOutput: string | null
}
```

**PDF layout (per D-06):**

Define styles using `StyleSheet.create({})` from @react-pdf/renderer:
- `page`: padding 40pt, fontFamily 'Helvetica' (built-in, no custom font registration needed for v1)
- `header`: marginBottom 20pt, borderBottomWidth 1, borderBottomColor '#e4e4e7', paddingBottom 10pt
- `title`: fontSize 18, fontWeight 'bold', color '#18181b'
- `subtitle`: fontSize 10, color '#71717a', marginTop 4pt
- `section`: marginTop 16pt, marginBottom 8pt
- `sectionTitle`: fontSize 12, fontWeight 'bold', color '#27272a', marginBottom 6pt
- `body`: fontSize 10, color '#3f3f46', lineHeight 1.5
- `fieldLabel`: fontSize 9, fontWeight 'bold', color '#71717a', textTransform 'uppercase', marginBottom 2pt
- `fieldValue`: fontSize 10, color '#18181b', marginBottom 10pt
- `footer`: position 'absolute', bottom 30, left 40, right 40, fontSize 8, color '#a1a1aa', textAlign 'center', borderTopWidth 1, borderTopColor '#e4e4e7', paddingTop 8pt

**Document structure:**
```
<Document>
  <Page size="A4" style={styles.page}>
    <!-- Header -->
    <View style={styles.header}>
      <Text style={styles.title}>Agency OS</Text>
      <Text style={styles.subtitle}>{clientName} -- {date}</Text>
    </View>

    <!-- Section -->
    <View style={styles.section}>
      <Text style={styles.sectionTitle}>{processName}</Text>
      <Text style={styles.subtitle}>{phaseName}</Text>
    </View>

    <!-- Body: Structured output or raw text -->
    {structuredOutput ? renderStructuredFields(structuredOutput) : renderRawText(rawOutput)}

    <!-- Footer -->
    <View style={styles.footer} fixed>
      <Text>Generated by Agency OS</Text>
    </View>
  </Page>
</Document>
```

**Rendering structured output as PDF:**
Create a helper function `renderStructuredFields(data: Record<string, unknown>)` that iterates over the object keys and renders:
- For each key: a `<Text style={styles.fieldLabel}>` with the key (replace underscores with spaces, capitalize).
- For each value:
  - string: `<Text style={styles.fieldValue}>{value}</Text>`
  - number/boolean: `<Text style={styles.fieldValue}>{String(value)}</Text>`
  - array of strings: Each item as `<Text style={styles.body}>  - {item}</Text>`
  - nested object: Recursively render key-value pairs with indentation (add paddingLeft 12pt)
  - other: `<Text style={styles.fieldValue}>{JSON.stringify(value, null, 2)}</Text>`

**Rendering raw text:**
`renderRawText(text: string | null)` returns `<Text style={styles.body}>{text ?? 'No output available'}</Text>`.

**Security (T-07-01 continuation):** All content is rendered via react-pdf `<Text>` elements which are inherently safe (no HTML injection possible in PDF rendering). No user-controlled content is used in style attributes.

**Do NOT:**
- Do NOT register custom fonts (v1 uses built-in Helvetica per Claude's Discretion).
- Do NOT implement batch export (deferred per CONTEXT.md).
