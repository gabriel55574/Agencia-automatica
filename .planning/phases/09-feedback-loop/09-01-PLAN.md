    ```sql
    -- Agency OS: Phase 9 - Feedback Loop
    -- reset_pipeline_cycle: Resets a client pipeline for a new cycle.
    -- Preserves all historical data (squad_jobs, gate_reviews, deliverables).

    CREATE OR REPLACE FUNCTION reset_pipeline_cycle(p_client_id UUID)
    RETURNS void LANGUAGE plpgsql AS $$
    DECLARE
      v_client clients%ROWTYPE;
      v_temp_phase_id UUID;
      v_phase_ids UUID[] := ARRAY[]::UUID[];
      v_phase_names TEXT[] := ARRAY[
        'Diagnostico',
        'Engenharia de Valor',
        'Go-to-Market',
        'Tracao e Vendas',
        'Retencao e Escala'
      ];
      i INT;
    BEGIN
      -- Lock client row to prevent concurrent resets
      SELECT * INTO v_client
      FROM clients
      WHERE id = p_client_id
      FOR UPDATE;

      IF NOT FOUND THEN
        RAISE EXCEPTION 'Client % not found', p_client_id;
      END IF;

      -- Guard: only reset active clients who completed Phase 5
      IF v_client.current_phase_number < 5 THEN
        RAISE EXCEPTION 'Client % has not completed Phase 5 (current phase: %)',
          p_client_id, v_client.current_phase_number;
      END IF;

      -- Check Phase 5 is completed
      IF NOT EXISTS (
        SELECT 1 FROM phases
        WHERE client_id = p_client_id
          AND phase_number = 5
          AND status = 'completed'
      ) THEN
        RAISE EXCEPTION 'Client % Phase 5 is not completed', p_client_id;
      END IF;

      -- Step 1: Increment cycle_number and reset current_phase_number to 1
      UPDATE clients
      SET cycle_number = cycle_number + 1,
          current_phase_number = 1
      WHERE id = p_client_id;

      -- Step 2: Mark all existing phases as completed (they belong to the old cycle)
      -- We do NOT delete them -- historical data is preserved.
      -- Instead, create NEW phase rows for the new cycle.
      FOR i IN 1..5 LOOP
        INSERT INTO phases (client_id, phase_number, name, status, started_at)
        VALUES (
          p_client_id, i, v_phase_names[i],
          CASE WHEN i = 1 THEN 'active'::TEXT ELSE 'pending'::TEXT END,
          CASE WHEN i = 1 THEN NOW() ELSE NULL END
        )
        RETURNING id INTO v_temp_phase_id;
        v_phase_ids := v_phase_ids || v_temp_phase_id;
      END LOOP;

      -- Step 3: Create new process rows for the new cycle
      INSERT INTO processes (phase_id, client_id, process_number, name, squad, status)
      VALUES
        (v_phase_ids[1], p_client_id, 1,  'Pesquisa de Mercado e Insights',    'estrategia',   'pending'),
        (v_phase_ids[1], p_client_id, 2,  'Segmentacao, Targeting e Personas', 'estrategia',   'pending'),
        (v_phase_ids[2], p_client_id, 3,  'Posicionamento',                    'estrategia',   'pending'),
        (v_phase_ids[2], p_client_id, 4,  'Grand Slam Offers',                 'estrategia',   'pending'),
        (v_phase_ids[2], p_client_id, 5,  'Pricing',                           'estrategia',   'pending'),
        (v_phase_ids[2], p_client_id, 6,  'Branding Estrategico',              'estrategia',   'pending'),
        (v_phase_ids[3], p_client_id, 7,  'Planejamento G-STIC',               'planejamento', 'pending'),
        (v_phase_ids[3], p_client_id, 8,  'Design de Canais de Distribuicao',  'planejamento', 'pending'),
        (v_phase_ids[3], p_client_id, 9,  'Varejo e Omnichannel',              'planejamento', 'pending'),
        (v_phase_ids[3], p_client_id, 10, 'Logistica e Supply Chain',          'planejamento', 'pending'),
        (v_phase_ids[3], p_client_id, 11, 'Marketing de Causa e RSC',          'planejamento', 'pending'),
        (v_phase_ids[4], p_client_id, 12, 'Producao Criativa',                 'growth',       'pending'),
        (v_phase_ids[4], p_client_id, 13, 'Comunicacao Integrada IMC',         'growth',       'pending'),
        (v_phase_ids[4], p_client_id, 14, 'Bullseye Framework - 19 Canais',    'growth',       'pending'),
        (v_phase_ids[4], p_client_id, 15, 'Funil de Vendas',                   'growth',       'pending'),
        (v_phase_ids[5], p_client_id, 16, 'CRM, Lealdade e CLV',               'crm',          'pending');

      -- Step 4: Create new quality gate rows
      INSERT INTO quality_gates (phase_id, client_id, gate_number, status)
      VALUES
        (v_phase_ids[1], p_client_id, 1, 'pending'),
        (v_phase_ids[2], p_client_id, 2, 'pending'),
        (v_phase_ids[3], p_client_id, 3, 'pending'),
        (v_phase_ids[4], p_client_id, 4, 'pending');
    END;
    $$;
    ```

    IMPORTANT: The `phases` table has a UNIQUE constraint on `(client_id, phase_number)`. This means we CANNOT insert new phase rows with the same phase_number for the same client. To handle this, the migration must FIRST drop the unique constraint on phases (and processes) and replace it with a composite unique that includes a cycle identifier, OR we take a different approach: delete the old phase/process/gate rows before creating new ones. Since the CONTEXT.md says "Preserves all previous cycle data (squad_jobs, gate_reviews remain)", and squad_jobs references phase_id, we need to handle the FK carefully.

    The safest approach: Before inserting new phases, DELETE the old phase rows. But this would cascade-delete processes and gates (ON DELETE CASCADE). Squad_jobs reference phase_id with no ON DELETE CASCADE (just a plain FK), so deletion would fail.

    REVISED APPROACH: Instead of creating new rows, RESET the existing phase/process/gate rows in-place:
    1. Update all phases: set status='pending' (except phase 1 which becomes 'active'), clear started_at/completed_at
    2. Update all processes: set status='pending', clear started_at/completed_at/output_json/output_markdown/input_snapshot
    3. Update all quality_gates: set status='pending', clear ai_review_json/checklist_results/operator_decision/operator_notes/reviewed_at
    4. This preserves the same phase_id/process_id UUIDs, so squad_jobs and gate_reviews still reference valid rows
    5. The old completed squad_jobs remain linked to the same process_ids -- they serve as the "previous cycle" data

    This approach is simpler and respects all FK constraints. The feedback extraction queries squad_jobs by client_id + phase_number=5 + status='completed', and the old completed jobs remain in the table.

    Use this RESET approach in the migration instead of creating new rows. Update the SQL accordingly.
