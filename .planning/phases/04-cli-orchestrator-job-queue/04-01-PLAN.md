```sql
-- claim_next_job() — DO NOT REWRITE. Worker calls supabase.rpc('claim_next_job')
CREATE OR REPLACE FUNCTION claim_next_job()
RETURNS SETOF squad_jobs AS $$
BEGIN
  RETURN QUERY
  UPDATE squad_jobs
  SET status = 'running', started_at = NOW()
  WHERE id = (
    SELECT id FROM squad_jobs
    WHERE status = 'queued'
    ORDER BY created_at ASC
    LIMIT 1
    FOR UPDATE SKIP LOCKED
  )
  RETURNING *;
END;
$$ LANGUAGE plpgsql;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker → Claude CLI | Worker spawns a child process with job.cli_command as the prompt argument |
| squad_jobs.cli_command → spawn args | Untrusted data from the database becomes a spawn argument |
| Claude CLI → external APIs | Claude CLI makes outbound API calls using ANTHROPIC_API_KEY |
| worker → Supabase DB | Worker writes progress_log and status updates back to squad_jobs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | squad_jobs.cli_command → spawn args | mitigate | cli_command is passed as a single positional string argument to `spawn('claude', [...args])`. Node.js `spawn` (not `exec`) does NOT invoke a shell — argument is passed directly to the claude binary as argv, making shell injection impossible. Never use `exec` or shell: true. Documented in action as a CRITICAL rule. |
| T-04-02 | Elevation of Privilege | Claude CLI permission-mode auto | mitigate | Use `--permission-mode auto` (auto-approves tool requests) rather than `--dangerously-skip-permissions`. This allows tool use but does not bypass all safety checks. The worker runs on a dedicated VPS with a scoped deployment directory. |
| T-04-03 | Denial of Service | Runaway child processes (no concurrency limit) | mitigate | MAX_CONCURRENT = 2 enforced in tryClaimAndRun() before every job claim. In-memory activeJobs Map tracks all spawned processes. SIGTERM handler kills all active processes on worker shutdown. |
| T-04-04 | Denial of Service | Job timeout / zombie processes | mitigate | 30-minute timeout enforced in recoverStuckJobs(). Stuck jobs are SIGTERM'd and removed from activeJobs. Heartbeat runs every 5 minutes and on worker startup. |
| T-04-05 | Information Disclosure | ANTHROPIC_API_KEY in environment | mitigate | Key set via VPS environment variable or PM2 env block (NOT in ecosystem.config.js committed to git). .gitignore must exclude .env files. ecosystem.config.js env block contains only NODE_ENV and PORT — no secrets. |
